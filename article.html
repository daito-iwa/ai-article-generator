システム構成

今回構築したシステムの全体像は以下の通りです：

Python 
import openai
import json
from typing import Dict, List

class ArticleGenerator:
    def __init__(self, api_key: str):
        openai.api_key = api_key
        self.system_prompt = """
        あなたは技術記事の専門ライターです。
        読みやすく、実用的で、SEOに配慮した記事を作成してください。
        """
    
    def generate_article(self, topic: str, keywords: List[str]) -> Dict:
        prompt = f"""
        テーマ: {topic}
        キーワード: {', '.join(keywords)}
        
        上記のテーマで2000文字程度の技術記事を作成してください。
        """
        
        response = openai.ChatCompletion.create(
            model="gpt-4",
            messages=[
                {"role": "system", "content": self.system_prompt},
                {"role": "user", "content": prompt}
            ],
            temperature=0.7
        )
        
        return {
            "content": response.choices[0].message.content,
            "usage": response.usage
        }

プロンプトエンジニアリングのコツ

高品質な記事を生成するために、以下のようなプロンプトの工夫を行いました：

1. システムプロンプトの活用

システムプロンプトでAIの役割を明確に定義することで、一貫性のある記事を生成できます。

プロのTIPS

システムプロンプトには「あなたは〜です」という形で役割を与え、期待する出力の形式も具体的に指示しましょう。

2. 温度設定の調整

temperatureパラメータを0.7に設定することで、創造性と一貫性のバランスを取りました。

  • 0.0-0.3: 一貫性重視、ファクトチェックが重要な記事
  • 0.4-0.7: バランス型、技術記事に最適
  • 0.8-1.0: 創造性重視、ブレインストーミング向け

API呼び出しのエラーハンドリングは必須です：

Python 
try:
    response = openai.ChatCompletion.create(...)
    return response.choices[0].message.content
except openai.error.RateLimitError:
    print("レート制限に達しました。しばらく待ってから再試行してください。")
except openai.error.APIError as e:
    print(f"OpenAI API エラー: {e}")
except Exception as e:
